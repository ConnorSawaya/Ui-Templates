# Template Guide

## Minimal Design Rules

- Start simple.
- Add only the UI needed to express the concept.
- Prefer whitespace, clear hierarchy, and restrained color.
- Avoid turning concept screens into dashboards unless the concept explicitly requires it.

## Originality Rules

- Do not copy real brands exactly.
- Do not reuse real mascots, logos, icons, or trademarked visual systems.
- Inspiration is allowed.
- Direct replication is not.

## Content Rules

- Keep content placeholder-only unless a template explicitly requires realistic sample data.
- Prefer labels like `Lesson 1`, `Unit 1`, `Option 1`, `Card Title`, and `Placeholder`.
- Do not add dense copy to make a concept look more finished than it is.

## Responsive Checklist

- Works on small mobile widths
- Core actions remain visible
- Tap targets remain comfortable
- Desktop preview stays centered and not overly wide
- Scrolling feels intentional and not broken

## Polish Checklist

- Naming is consistent
- States are visually clear
- Empty/loading/reward states are handled if relevant
- Animations are subtle
- Borders, spacing, and shadows feel intentional
- README, metadata, screenshots, and notes are present
